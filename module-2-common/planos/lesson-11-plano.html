Resumo

A aula 11 conecta a stack à internet real. Os alunos atravessam o modelo TCP/IP de 4 camadas, comparam TCP × UDP com 3-way handshake, percorrem o DNS registro por registro, entendem o TLS handshake conceitual, exploram HTTP/1.1 × 2 × 3, dominam cache HTTP (Cache-Control, ETag, 304), atravessam uma CDN (origem/edge/PoP) e fecham com padrões de tolerância a falhas (retry com backoff exponencial + jitter, timeout, idempotência, circuit breaker, fallback, health checks, observabilidade RED).

O encerramento amarra tudo no nosso stack TS + Express + EJS + SSR + pg — a infraestrutura de rede deixa de ser caixa-preta para virar superfície de design.

Objetivos de aprendizagem

  • OA-1Posicionar HTTP, TLS, TCP e IP nas camadas TCP/IP de 4 níveis.
  • OA-2Distinguir TCP de UDP e justificar a escolha conforme o caso de uso.
  • OA-3Resolver um nome via DNS passo a passo, identificando registros A / CNAME / MX / TXT.
  • OA-4Aplicar cache HTTP correto (Cache-Control, ETag, 304) por tipo de recurso.
  • OA-5Desenhar a interação navegador → CDN → origem com hit/miss explícitos.
  • OA-6Aplicar retry com backoff+jitter, timeout, idempotência e circuit breaker no nosso stack TS.

Pré-requisitos

  • Aulas 1–10 concluídas: o projeto já está em produção (ou pronto para deploy) com testes e CI verdes.
  • Domínio próprio do projeto acessível (mesmo que ainda em ambiente de staging).
  • Ambiente local com dig ou nslookup disponível e acesso ao DevTools do navegador.
Importante

Aqui a equipe blinda o projeto contra falhas reais de rede — não é teoria abstrata: cada conceito vira mudança de código no projeto integrador antes das 16h.

TalkPor que um dev backend precisa entender redes

Casos reais: timeout sem retry derrubando produção, cache mal configurado mostrando dado errado para outro usuário, falta de idempotência cobrando duas vezes. Rede não é problema do "ops", é design de API.

Estratégias de metodologia ativa

  • Simulação de falha — o professor "mata" um endpoint da CDN ou da origem; cada equipe diagnostica pelo comportamento (status, headers, latência) qual camada falhou.
  • Competição de cache — em rodadas rápidas, qual time acerta o Cache-Control ideal para cada rota (HTML SSR, estáticos, JSON público, JSON privado, redirect)?
  • Debate retry seguro × perigoso — turma classifica chamadas reais do projeto em "pode dar retry" vs "nunca dar retry sem Idempotency-Key", justificando.

Encerramento do módulo

A aula 11 fecha o Módulo 2. Em vez de uma "próxima aula", esta seção amarra a trajetória: o módulo começou em HTML/CSS/JS, passou por banco de dados relacional e SQL, construiu um back-end TypeScript com Express + EJS em SSR sobre arquitetura MVC de 6 camadas, cobriu o sistema com testes e CI e agora blindou contra falhas de rede.

Retrospectiva das 11 aulas

  • Aulas 1–4 · Front estático e fundação relacional: HTML/CSS/JS, modelagem, SQL, JOINs.
  • Aulas 5–8 · Back-end MVC: Node.js + Express, controllers, services, repositories, helpers e EJS+SSR.
  • Aulas 9–10 · Qualidade: tipagem TS estrita, testes por camada, Husky e CI verde.
  • Aula 11 · Infraestrutura de rede: cache, CDN, TLS, resiliência e idempotência.

Ponte para os próximos módulos

O Módulo 3 (e os módulos seguintes do ciclo) parte deste projeto integrador como base — o stack TS + Express + EJS + SSR + pg permanece, e os próximos módulos vão adicionar autenticação, observabilidade, deploy avançado e arquiteturas distribuídas.

Checklist final para entrega do projeto integrador

  • DER + minimundo + dicionário de dados publicados em docs/banco/.
  • CRUD funcional em todas as entidades principais via Express + EJS (SSR).
  • Camadas MVC respeitadas (controllers / services / repositories / helpers / views).
  • Cobertura de testes mínima por camada + workflow GitHub Actions verde.
  • Cache + timeout + retry aplicados; docs/redes/resiliencia.md presente.
  • README com instruções claras de execução local + deploy.
Encerramento

Você sai do Módulo 2 sabendo construir um sistema web inteiro — do banco até o cache da CDN — com qualidade testável e tolerância a falhas. É a fundação sobre a qual todo o resto do ciclo será construído.
