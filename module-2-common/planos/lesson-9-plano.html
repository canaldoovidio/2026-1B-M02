Resumo

A aula 9 fecha o bloco de front-end. CSS moderno entra em escala: CSS Variables com tematização (claro/escuro), Flexbox e Grid para layout, mobile-first com clamp(), transitions performáticas animando apenas transform e opacity. A organização ganha BEM (.card__title, .card--featured) — convenções que tornam o CSS legível por toda a equipe.

No JS, salto de nível: ES Modules nativos no browser (<script type="module">), destructuring/spread/rest, optional chaining, classes ES, closures, persistência com localStorage, envio de forms com FormData + fetch() e delegação de eventos. Tudo aplicado ao projeto integrador.

Objetivos de aprendizagem

  • OA-1Aplicar CSS Variables com escopo dinâmico (tematização claro/escuro via atributo data-theme).
  • OA-2Compor layouts com Flexbox e Grid responsivos seguindo a abordagem mobile-first.
  • OA-3Estruturar CSS com BEM em componentes reutilizáveis (block__element--modifier).
  • OA-4Usar ES Modules nativos no browser (<script type="module">, import/export).
  • OA-5Manipular dados com destructuring/spread, classes ES e closures.
  • OA-6Persistir preferências do usuário com localStorage e enviar formulários via FormData + fetch().

Pré-requisitos

  • Aula 7 concluída: SSR com EJS, partials e DOM funcionando no projeto.
  • Aula 8 concluída: pelo menos 1 interação dinâmica com fetch() + estados loading/empty/error.
  • Repositório com public/css/ e public/js/ servidos por express.static.

Estratégias de metodologia ativa

  • Debate "BEM vs utility-first" — discussão guiada com argumentos de ambos os lados antes de o professor sintetizar quando cada abordagem cabe melhor.

Critério mínimo

Aluno que sair às 16h sem tema escuro persistido ou sem ao menos 1 componente em BEM está em débito técnico para a aula 10 — testes de UI dependem de marcações estáveis e previsíveis.

Conexão com a aula 10

Toda a stack está pronta: banco, back TypeScript em 6 camadas, SSR com EJS, fetch e CSS profissional. A aula 10 mostra como garantir que ela continua certa: testes por camada (helpers, services com mock, repositories com banco efêmero, controllers com supertest) e automação com Husky + GitHub Actions.

  • O design system congelado hoje permite testes visuais estáveis amanhã.
  • Os ES Modules da aula 9 facilitam a injeção de mocks no Jest.
Para o autoestudo da aula 10
